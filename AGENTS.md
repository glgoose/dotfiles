# dotfiles — agent conventions

## bin/ scripts

- All scripts start with `set -euo pipefail`.
- Secrets are stored in macOS Keychain via `security add-generic-password` and exported in `~/.zshrc`.
- Python CLI tools: use `uvx <tool-name>` (e.g. `uvx mlx-whisper`).
- Python libraries in inline scripts: use `uv run --with <pkg> python3 -`.
- Never require `pip install` or `uv tool install` — only `uv` itself needs to be present.
- For every long flag, also define a short flag: e.g. `--rename, -r`, `--dry-run, -n`, `--force, -f`. Keep short flags unique within the command. Prefer conventional shorthands where they exist.

## Zotero Actions & Tags scripts

When writing or editing a Zotero Actions & Tags custom script, use the `zotero-scripting` skill.

- When calling external processes via `subprocess`, guard the result: `(result || '').trim()` — subprocess can return `undefined` if the process produces no stdout.

## Citations

When you encounter a citation key in the format `Author####` (e.g., `Stone2004`, `Smith2021`), use the `zotero-lookup` skill to resolve it. The skill will fetch bibliographic data and context from Zotero.

## Git commits

- One commit per script or config file — never bundle unrelated files.
- Commit message describes what the script does differently, not just "update X".

## Agent skills layout

Skills follow the [agentskills.io](https://agentskills.io/specification) spec and live in one canonical store, bridged to each agent's expected path via stow.

- Canonical store: `agents/.agents/skills/<skill-name>/` (stowed → `~/.agents/skills/`).
  - Your own skills are real directories tracked here.
  - Vendor skills are symlinks pointing into `skills-vendor/<bundle>/...`.
- Vendor bundles: git submodules under `agents/.agents/skills-vendor/`.
  - `mattpocock-skills` → github.com/mattpocock/skills
  - `obsidian-skills` → github.com/kepano/obsidian-skills
  - Update both: `make update-skills` (bumps submodules, regenerates symlinks, commits).
- Vendor symlinks regenerated by `bin/sync-vendor-skills`. Per-vendor source paths and exclude lists are configured at the top of that script.
- Bridges (single-symlink, agent-agnostic):
  - `claude/.claude/skills` → `../../agents/.agents/skills`
  - `pi/.pi/skills` and `pi/.pi/agent/skills` → `../../agents/.agents/skills` (and `../../../...` for nested)
  - `copilot/.copilot/skills` → `../../agents/.agents/skills`

When creating a new skill (yours, not vendored): `mkdir agents/.agents/skills/<name>/`, write `SKILL.md`, then commit. Stow tree-folding makes it visible to all agents automatically.

Never edit files inside `agents/.agents/skills-vendor/` — those are submodules. Forking a vendor skill: copy it out into `agents/.agents/skills/<name>/`, add the original name to the exclude list in `bin/sync-vendor-skills`.
